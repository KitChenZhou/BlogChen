python基础

python基础

1. 可迭代对象，迭代器， 生成器，装饰器

  1. 迭代器，是访问集合元素的一种方式，特点：

    • 每次调用next()方法只访问一个元素，而且不能后退，便于循环比较大的数据集合，节省内存
    • 只能从头到尾访问，不能随机访问某个值
    • 迭代器提供了一个同意的访问集合的接口，只要定义了iter()方法对象，就可以使用迭代器访问

  2. 生成器，一个调用返回迭代器的函数，叫生成器，函数中包含yield语法，这个函数就会变成生成器

    • 生成器表达式：用来生成有规律的生成器

    • yield创建生成器

    • 生成器实现单线程的异步并发效果

    • 生成器中的send()方法使用

  3. 装饰器，对已有函数，在不改变它内在封装上扩展它的功能

    • 使用方式：

      • 先定义一个装饰器
      • 在定义你的业务函数或者类
      • 最后把这装饰器扣在函数上

2. python2和python3的区别，如何实现python2代码迁移到python3环境，编码方式有什么差别

3. 浅拷贝和深拷贝

  1. 浅拷贝：只复制了最外层的容器，副本中的元素是原容器中元素的引用，如果所有的元素都是不可变的， 那么没什么问题，但是如果有可变的元素，可能就会导致意想不到的问题。比如a中列表，b浅拷贝a，然后修改列表中的数会导致a中列表也会发生变化

    1、b = a: 赋值引用，a 和 b 都指向同一个对象。

    2、b = a.copy(): 浅拷贝, a 和 b 是一个独立的对象，但他们的子对象还是指向统一对象（是引用）。

    b = copy.deepcopy(a): 深度拷贝, a 和 b 完全拷贝了父对象及其子对象，两者是完全独立的。

  2. 深拷贝：即副本不共享内部对象的引用

数据库(Mysql)、缓存(Redis)、消息队列(Kafka)

4. @classmethod 和 @staticmethod的作用和区别

  1. @classmethod: 当我们需要和类直接进行交互，而不需要和实例进行交互时，类方法是最好的选择，类方法和实例方法类似，但是传递的不是类的实例，而是类本身，第一个参数是cls。我们可以用类的实例调用类方法，也可以直接用类名来调用
  2. @staticmethod: 静态方法类似普通方法，参数里面不用self，这些方法和类相关，但是又不需要类和实例中的任何消息、属性等，如果把这些方法写到外面，这样就把和类相关的代码分散到类外，使得之后对代码的理解和维护都是巨大的障碍，而静态方法就是用来解决这一类问题的
